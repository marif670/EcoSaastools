1. Introduction to WordPress

Zero-knowledge friendly: what WordPress is, differences between .org and .com, and main use-cases.

Definition: WordPress is an open-source content management system used to build websites without having to code every page.

Why WordPress is #1 CMS

  • Ease of use, huge ecosystem, extensibility, low cost.
  • Large community and marketplace of plugins/themes.

Mermaid Diagram â€” high-level site components

flowchart LR Visitor -->|requests| Server["Web Server"] Server --> WP["WordPress Core"] WP --> Theme["Theme (appearance)"] WP --> Plugin["Plugins (features)"] WP --> DB(("Database - MySQL/MariaDB")) WP --> Media["Media Files"]

WordPress.org vs WordPress.com (in detail)

WordPress.org (self-hosted) â€” full control: choose host, install any plugin/theme, customize code. Requires maintenance (updates, backups, security).

WordPress.com (hosted) â€” simpler setup, limited plugin/theme choice on free plans, paid tiers add features; hosting and maintenance handled by provider.

Use cases

  • Blogging & publishing
  • Business websites & landing pages
  • Ecommerce (WooCommerce)
  • Portfolios, memberships, LMS
Exercise 1.1: Write 3 sentences describing the site you want to build with WordPress.
Summary: WordPress is flexible â€” pick .org for full control or .com for managed convenience.
Beginner Mistakes to Avoid
  • Confusing .org and .com
  • Choosing theme before planning content

4. Security During Installation

Strong usernames & passwords

Use unique admin username and complex password. Use a password manager.

Mermaid Diagram â€” security checklist flow

graph TD Start[Start Install] --> UniqueUser{Choose admin username?} UniqueUser -->|yes| StrongPW[Use strong password] UniqueUser -->|no| Fail1[Change 'admin' -> set unique] StrongPW --> Prefix{Change DB prefix from wp_?} Prefix -->|yes| SSL[Enable SSL] Prefix -->|no| Warn[Increase other protections] SSL --> Finish[Complete install with secure defaults]

Default table prefix risks

Change wp_ to reduce automated exploit success rates.

Exercise 4.1: During install, set a non-default DB prefix and record it in your secure notes.
Summary: Early secure choices (username, password, prefix, SSL) greatly reduce risk.
Beginner Mistakes to Avoid
  • Leaving file editing enabled in production
  • Using 'admin' as username
